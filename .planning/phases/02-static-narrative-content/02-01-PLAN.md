Create `lib/relative-time.ts` with two exported functions:

```typescript
export function getRelativeTime(dateString: string): string
```
Returns human-readable relative time: "just now" (<1hr), "3 hours ago" (1-23hr), "2 days ago" (1-6d), "1 week ago" (7-29d), "2 months ago" (30d+). Uses integer math on Date.now() - new Date(dateString).getTime().

```typescript
export function isStale(dateString: string, thresholdDays: number): boolean
```
Returns true if dateString is more than thresholdDays old.

Create `lib/github.ts` with:

```typescript
export interface CommitInfo {
  repo: string
  date: string
  relativeTime: string
}

export async function getLatestCommit(): Promise<CommitInfo | null>
```

Implementation:
1. Read `process.env.SIGNAL_REPOS` (comma-separated, e.g. "quartermint/openefb,quartermint/v2cf,sternryan/msgvault").
2. If no env var or empty, return null.
3. For each repo, fetch `https://api.github.com/repos/${repo.trim()}/commits?per_page=1` with `{ next: { revalidate: 3600 } }` for ISR caching.
4. Find the most recent commit across all repos by comparing dates.
5. If the most recent commit is >14 days old (use `isStale(date, 14)`), check `process.env.FALLBACK_SIGNAL`. If FALLBACK_SIGNAL exists, return `{ repo: 'fallback', date: '', relativeTime: process.env.FALLBACK_SIGNAL }`.
6. If no FALLBACK_SIGNAL, return null.
7. Wrap entire function in try/catch. On any error, attempt FALLBACK_SIGNAL, then return null.
8. For the happy path, return `{ repo: repoName (just the repo part after /), date: isoString, relativeTime: getRelativeTime(isoString) }`.

Create `__tests__/living-signal.test.ts`:
- Test `getRelativeTime` with known date offsets: just now, hours, days, weeks, months.
- Test `isStale` returns true/false correctly for 14-day threshold.
- Test that `getLatestCommit` returns null when SIGNAL_REPOS is not set (mock process.env).
- Tests use vitest `describe/it/expect` pattern. For getRelativeTime tests, compute dates relative to Date.now() and pass ISO strings.

**3. Create `components/section-wrapper.tsx` (per NAR-07, ENG-03):**
- Add `'use client'` directive at top.
- Props: `{ children: ReactNode, bg: 'bg' | 'surface', label: string }`.
- Renders a `<section>` element with `aria-label={label}`.
- Background: `bg === 'bg' ? 'bg-bg' : 'bg-surface'` (use conditional className, not template literal with dynamic segment, because Tailwind v4 needs full class names for scanning).
- Inner content wrapper: `<div className="max-w-[var(--spacing-content-max)] mx-auto px-4 sm:px-6 lg:px-8 py-[var(--spacing-section-mobile)] lg:py-[var(--spacing-section-desktop)]">`.
- Entrance animation: Use IntersectionObserver with `threshold: 0.3`, fire once per section (`observer.unobserve(el)` after first intersection).
- Transition classes: `transition-all duration-[400ms] ease-out`. When not visible: `opacity-0 translate-y-2`. When visible: `opacity-100 translate-y-0`.
- **Critical (Pitfall 4):** Accept a `noAnimation` prop (boolean, default false). When true, render immediately visible without IntersectionObserver (for above-the-fold sections like Hero). The Hero section passes `noAnimation={true}`.
- `translate-y-2` is Tailwind's 8px (0.5rem = 8px), matching the ENG-03 spec.

**4. Create `components/living-signal.tsx` (per ENG-04):**
- Add `'use client'` directive at top.
- Props: `{ signal: { repo: string, date: string, relativeTime: string } | null }`.
- If signal is null, render nothing (return null).
- If signal.repo is 'fallback', render: `<p>` with signal.relativeTime text (the FALLBACK_SIGNAL value). DM Sans 14px/400, `text-text-faint`.
- If signal.repo is a real repo: render "Last shipped to {repo} {relativeTime}" in DM Sans 14px/400, `text-text-muted`. If `isStale(signal.date, 7)` is true, use `text-text-faint` instead.
- Green dot indicator: a 6px circle (`w-1.5 h-1.5 rounded-full bg-accent inline-block mr-2`) before the text when NOT stale and NOT fallback.
- Fade-in animation: On mount, use a `useState(false)` + `useEffect(() => setVisible(true), [])` pattern. Apply `transition-opacity duration-[600ms] ease-in` with 300ms delay. When not visible: `opacity-0`. When visible: `opacity-100`. The 300ms delay uses `delay-300` Tailwind class.
- `prefers-reduced-motion` is handled globally by the CSS rule in globals.css (sets transition-duration to near-zero), so no additional check needed in the component.
- Screen reader: wrap text in a `<span>` and add an `<span className="sr-only">` prefix: "Last code pushed to GitHub" for accessibility context.
