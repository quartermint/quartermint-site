# Phase 1 — UI Design Contract

> Visual and interaction contract for the foundation and design system phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (custom components with Tailwind utilities) |
| Icon library | none (GitHub SVG inline only — single icon used for public repo links) |
| Font | Instrument Serif (display) + DM Sans (body) via `next/font/google` |

**Rationale:** CLAUDE.md explicitly rejects shadcn/ui as "overkill for a narrative site with editorial typography." The design system is custom — CSS custom properties defined via Tailwind v4 `@theme` directive, which creates both CSS variables and utility classes from a single source. No component library abstraction layer.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon gaps, badge letter-spacing (1px is an exception for letter-spacing only) |
| sm | 8px | Compact element spacing, chip horizontal padding, section entrance translateY |
| md | 16px | Default element spacing, component vertical gap, chip horizontal padding, shelf gap |
| lg | 24px | Card padding, card gap (featured grid), chat heading-to-chips gap |
| xl | 28px | Major element vertical gap (between major content blocks within sections) |
| 2xl | 32px | Featured system row gap, layout-level spacing |
| 3xl | 48px | Section vertical padding (mobile), nav height, chat input height, touch target min |
| 4xl | 80px | Section vertical padding (desktop) |

Exceptions:
- **44px** minimum touch target (WCAG requirement, not a spacing token — applied as `min-width`/`min-height` on interactive elements)
- **28px** is not a multiple of 8 but IS a multiple of 4 — used specifically for "between major elements" vertical gap per design doc
- **6px** border-radius on buttons, chip border-radius — design doc spec, used as component property not spacing token
- **8px** border-radius on cards and containers — component property
- **2px** focus ring width + offset — accessibility property

### Color Distribution (60/30/10)

| Role | Share | Elements |
|------|-------|----------|
| Dominant (60%) | `--color-bg` | Page background, white alternating sections, chat container background, /invest page background |
| Secondary (30%) | `--color-surface` | Mint alternating sections, card backgrounds, chip backgrounds, shelf hover states |
| Accent (10%) | `--color-accent` | Reserved ONLY for: focus rings (2px solid), living signal indicator, chat typing dots, /invest thesis pull-quote left border (3px), section labels |

**Accent is decorative only.** Never used for text that must be readable. No destructive semantic color needed in this phase (no delete actions, no form submissions, no data mutation).

## `lib/systems.ts` Schema

```typescript
interface System {
  name: string;            // Display name (e.g., "LifeVault")
  slug: string;            // URL slug (e.g., "lifevault")
  oneLiner: string;        // Short description
  problem: string;         // Empathy-first problem framing
  solution: string;        // What the system does
  techBadge: string;       // One of: "Go", "Swift", "TypeScript", "Python", "Raspberry Pi", "Go + Swift"
  isPublic: boolean;       // Has public GitHub repo
  githubUrl: string | null; // Full GitHub URL or null
  status: "active" | "paused"; // Current activity status
  featured: boolean;       // One of the 4 featured systems
}
```

**15 entries:** 4 featured (LifeVault, Relay, OpenEFB, v2cf) + 11 shelf (msgvault, pixvault, drivevault, Mainline, Mission Control, Sovereign Flight Recorder, foundry, skygate, open-ez, ticker, signal-glass).

Featured systems have locked problem/solution copy from the design doc. Shelf system one-liners to be drafted by Claude and approved by user before shipping (per CONTEXT.md D-05).

## Tailwind v4 `@theme` Token Structure

All design tokens are defined in `app/globals.css` using the `@theme` directive. Tailwind v4 automatically generates both CSS custom properties and utility classes from these definitions.

```css
/* Token structure — exact values above, this shows organization */
@theme {
  /* Colors (consumed via var(--color-*) and Tailwind utilities) */
  --color-bg: #FFFFFF;
  --color-surface: #E6F4F1;
  --color-accent: #A8E6CF;
  --color-text: #333A45;
  --color-text-muted: #555555;
  --color-text-faint: #888888;

  /* Typography (font families set via next/font CSS variables) */
  --font-display: var(--font-instrument-serif);
  --font-body: var(--font-dm-sans);

  /* Spacing (consumed via Tailwind spacing utilities) */
  --spacing-section-desktop: 80px;
  --spacing-section-mobile: 48px;
  --spacing-content-max: 1120px;
  --spacing-invest-max: 680px;
}
```

Dark mode via media query on `:root`, NOT a class toggle:

```css
@media (prefers-color-scheme: dark) {
  :root {
    --color-bg: #1A1D23;
    --color-surface: #232830;
    --color-accent: #7CCFB0;
    --color-text: #E8ECF0;
    --color-text-muted: #A0A8B4;
    --color-text-faint: #6B7280;
  }
}
```
