<objective>
Implement returning visitor detection so the chat remembers who has visited before and personalizes greetings based on recency and conversation topics, with server-side session tracking via HttpOnly cookie and Upstash Redis.

Purpose: Visitors feel recognized without being tracked invasively. The "barista who remembers your order" experience creates engagement continuity across visits.
Output: rv cookie issuance via proxy.ts, visitor state CRUD in Redis, greeting component with tier-based personalization, background topic extraction via Haiku.
</objective>

    Actually, use `crypto.randomUUID()` (native, per D-03 and RESEARCH Don't Hand-Roll):

    ```typescript
    import type { NextRequest } from 'next/server'
    import { NextResponse } from 'next/server'

    export function middleware(request: NextRequest) {
      const response = NextResponse.next()

      // Set rv (returning visitor) cookie on first visit (per D-03)
      if (!request.cookies.get('rv')) {
        const visitorId = crypto.randomUUID()
        response.cookies.set('rv', visitorId, {
          httpOnly: true,
          sameSite: 'strict',
          secure: process.env.NODE_ENV === 'production',
          maxAge: 90 * 24 * 60 * 60, // 90 days
          path: '/',
        })
      }

      return response
    }

    export const config = {
      matcher: ['/((?!api|_next/static|_next/image|favicon.ico).*)'],
    }
    ```
