# Phase 3 -- UI Design Contract

> Visual and interaction contract for the embedded AI chat system. Generated by gsd-ui-researcher, verified by gsd-ui-checker. Inherits design system from Phase 1/2. Phase 3 replaces the ChatPlaceholder component with a fully functional streaming chat interface.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (custom components with Tailwind v4 utilities) |
| Icon library | none (no new icons -- send arrow is a text button) |
| Font | Instrument Serif (display) + DM Sans (body) via `next/font/google` |

**Inherited from Phase 1/2.** CLAUDE.md explicitly rejects shadcn/ui. All styling through Tailwind v4 `@theme` inline tokens referencing CSS custom properties. Phase 3 builds the chat interface components using the established token system.

## Spacing Scale

Inherited from Phase 1/2 UI-SPEC. All values are multiples of 4.

| Token | Value | Phase 3 Usage |
|-------|-------|---------------|
| xs | 4px | Gap between typing dots, inline icon gaps |
| sm | 8px | Chat message vertical gap within a run, starter chip internal vertical padding, input field internal horizontal padding adjustment |
| md | 16px | Gap between heading and starter chips, gap between messages of different roles, chat container horizontal padding (mobile), privacy notice top margin |
| lg | 24px | Gap between starter chips row and input field, chat container internal padding (desktop), gap between message bubbles and input area |
| 2xl | 32px | Chat container top/bottom padding on desktop |
| 3xl | 48px | Input field height, mobile overlay header height |
| 4xl | 64px | Not used in Phase 3 |

Layout constants (inherited, not spacing tokens):
- **`--spacing-content-max: 1120px`** -- Page content max-width (chat section lives within this)
- **`--chat-max-width: 720px`** -- Chat container max-width (from CHAT-06, design doc)

Phase 3 exceptions:
- **44px** minimum touch target (WCAG, applied to send button, starter chips, close button, all tappable elements)
- **8px** border-radius on chat container, input field, mobile overlay
- **6px** border-radius on starter chips (design doc spec)
- **2px** focus ring width + offset (inherited accessibility baseline)

|-------|-------|---------------|
| `--color-bg` | #FFFFFF | Chat container background, chat section background (white alternating section), input field background |
| `--color-surface` | #E6F4F1 | Assistant message bubble background, starter chip background |
| `--color-accent` | #A8E6CF | Typing animation dots, focus ring on input field |

| `--color-text-muted` | #555555 | Input placeholder text, rate limit body copy |
| `--color-text-faint` | #888888 | Privacy notice text, input field border (1px solid), "Thinking..." label, starter chip text |

### Dark Mode (`@media (prefers-color-scheme: dark)`)

Inherited from Phase 1 -- all tokens swap automatically via CSS custom properties. No additional dark mode work in Phase 3 beyond verifying contrast.

### Color Distribution (60/30/10)

| Role | Share | Phase 3 Elements |
|------|-------|------------------|
| Dominant (60%) | `--color-bg` | Chat container background, input field background, chat section background, mobile overlay background |
| Secondary (30%) | `--color-surface` | Assistant message bubbles, starter chip backgrounds |
| Accent (10%) | `--color-accent` | Reserved ONLY for: typing animation dots (3 dots, pulsing), focus ring on chat input field |

### Chat-Specific Color Assignments

| Element | Background | Text Color | Border |
|---------|-----------|------------|--------|
| Chat container | `--color-bg` | -- | none (shadow provides containment) |
| User message bubble | `--color-text` (#333A45) | `--color-bg` (#FFFFFF) | none |
| Assistant message bubble | `--color-surface` (#E6F4F1) | `--color-text` (#333A45) | none |
| Starter chip | `--color-surface` (#E6F4F1) | `--color-text-faint` (#888888) | none |
| Starter chip hover | `--color-surface` with slight opacity shift | `--color-text` (#333A45) | none |
| Input field | `--color-bg` (#FFFFFF) | `--color-text` (#333A45) | 1px solid `--color-text-faint` |
| Input field focus | `--color-bg` (#FFFFFF) | `--color-text` (#333A45) | 1px solid `--color-accent` + focus ring |
| Send button | `--color-text` (#333A45) | `--color-bg` (#FFFFFF) | none |
| Send button disabled | `--color-text` at 40% opacity | `--color-bg` (#FFFFFF) | none |
| Privacy notice | transparent | `--color-text-faint` (#888888) | none |
| Error state container | transparent | `--color-text-muted` (#555555) | none |
| Rate limit state | transparent | `--color-text-muted` (#555555) | none |

### Contrast Verification (Phase 3 specific)

| Combination | Ratio | Grade | Context |
|-------------|-------|-------|---------|
| `--color-bg` (#FFFFFF) on `--color-text` (#333A45) | 10.5:1 | AAA | User message text on dark bubble |
| `--color-text` (#333A45) on `--color-surface` (#E6F4F1) | 8.2:1 | AAA | Assistant message text on mint bubble |
| `--color-text-faint` (#888888) on `--color-surface` (#E6F4F1) | ~2.8:1 | AA Large | Starter chip text on mint chip (14px, meets threshold) |
| `--color-text-faint` (#888888) on `--color-bg` (#FFFFFF) | 3.5:1 | AA Large | Privacy notice text |
| `--color-text-muted` (#555555) on `--color-bg` (#FFFFFF) | 7.5:1 | AAA | Error/rate limit copy |

**No destructive semantic color.** Phase 3 has no delete actions. Error states use `--color-text-muted` text on `--color-bg` background -- friendly copy, not alarming red.

### Rate Limit Copy

| State | Copy | CTA | Source |

### Privacy Notice Copy

| Element | Copy | Source |
|---------|------|--------|
| Privacy notice below input | "Messages are logged. [Privacy policy](/privacy)" | CHAT-04 -- DM Sans 12px in spec, using 14px Caption role for readability at AA Large contrast. Links to /privacy. |

**Privacy notice sizing:** CHAT-04 specifies 12px. However, 12px text at `--color-text-faint` on `--color-bg` achieves 3.5:1 contrast -- this only meets WCAG AA Large for text >= 14px bold or 18px regular. At 12px regular, it falls below AA. Implementation uses 14px (Caption role) at `--color-text-faint` to maintain the Phase 1 contrast commitment. The visual diminishment from `--color-text-faint` color achieves the spec's intent of "small, unobtrusive notice."

### Mobile Overlay Copy

|---------|------|--------|
| Mobile overlay header | No text header -- same chat heading inside the overlay content | Design doc mobile state |
| Close/back action | "Close" button or back arrow | Design doc: "Back button or swipe-down closes" |

### Internal Layout (top to bottom, idle state)

```
[  Heading: "Ask me anything about what I'm building"  ]  -- Instrument Serif 24px
[  16px gap                                             ]
```

## Chat Interaction States (CHAT-05)

### State: Idle (initial)

| Element | Spec |
|---------|------|
| Heading | Instrument Serif 24px/400/1.2, `--color-text`, centered |
| Starter chips | 3 horizontal chips, wrap on mobile. DM Sans 14px/400, `--color-text-faint` text, `--color-surface` background, 6px border-radius, 8px vertical / 16px horizontal padding, 8px gap between chips. Min-height 44px (touch target). |
| Starter chip hover | Text shifts to `--color-text`, subtle transition 150ms ease |
| Input field | Full width, 48px height, `--color-bg` background, 1px solid `--color-text-faint` border, 8px border-radius, DM Sans 16px, placeholder "Ask me anything..." in `--color-text-muted`. Focus: border changes to `--color-accent` + focus ring. |
| Send button | Inline right of input (or at right edge inside input area). DM Sans 14px/600, `--color-text` background, `--color-bg` text, 6px border-radius, min-width 44px, min-height 44px. Disabled when input is empty or loading (40% opacity). |

### State: Active (conversation in progress)

| Element | Spec |
|---------|------|
| Heading | Hidden (removed from DOM after first user message) |
| Starter chips | Hidden (removed from DOM after first user message) |
| Message area | Scrollable container, flex-grow, auto-scroll to bottom on new message |
| User message | Right-aligned, max-width 80%, `--color-text` background, `--color-bg` text, DM Sans 16px/400/1.7, 16px horizontal / 12px vertical padding, `border-radius: 16px 16px 4px 16px` (flat bottom-right corner for user) |
| Assistant message | Left-aligned, max-width 80%, `--color-surface` background, `--color-text` text, DM Sans 16px/400/1.7, 16px horizontal / 12px vertical padding, `border-radius: 16px 16px 16px 4px` (flat bottom-left for assistant). Content rendered via `react-markdown` + `remark-gfm`. |
| Message gap | 16px between messages of different roles, 8px between consecutive messages from same role |
| Input | Same as idle state. Returns focus after assistant response completes. |

### State: Typing (AI responding)

| Element | Spec |
|---------|------|
| User message | Rendered immediately (right-aligned, as above) |
| Typing indicator | Left-aligned, same position as assistant bubble. Three dots in `--color-accent`, 8px diameter each, 4px gap between dots. Pulsing animation: sequential opacity 0.3 -> 1.0 -> 0.3, staggered 200ms per dot, total cycle 600ms. "Thinking..." label: DM Sans 14px/400, `--color-text-faint`, 8px left of dots. |
| Input | Disabled while loading (reduced opacity on send button, input still visible for reading) |
| Streaming | Once first token arrives, typing indicator is replaced by the assistant bubble. Response streams in token-by-token (word-by-word appearance via `useChat` streaming). |

### State: Error (API failure / Upstash unreachable)

| Element | Spec |
|---------|------|
| Error message | Replaces typing indicator position. DM Sans 16px/400, `--color-text-muted`. No alarming colors, no stack traces. |
| Copy | "I'm not available right now. Email Ryan directly." |
| CTA | `mailto:ryan@quartermint.com` on "Email Ryan directly" -- styled as inline text link with underline, `--color-text` color. |
| Input | Remains enabled so visitor can retry. |

### State: Rate Limit (session or IP cap)

| Element | Spec |
|---------|------|
| Rate limit message | Appears in message area as an assistant-style bubble. `--color-surface` background, same border-radius as assistant message. |
| Copy (session) | "Great questions. For a deeper conversation:" |
| Copy (IP/429) | "You've been busy. Let's continue over a call:" |
| CTA | "Book a call" -- styled as inline text link with underline, links to Google Calendar booking URL (same as contact section). `--color-text` color. |
| Input | Disabled with `disabled` attribute. Placeholder changes to "Message limit reached" in `--color-text-faint`. |

### State: Mobile (full-screen overlay)

| Element | Spec |
|---------|------|
| Trigger | "Ask me anything" hero CTA on viewports < 640px. Instead of smooth-scrolling to `#chat-section`, opens the mobile overlay. |
| Container | `position: fixed; inset: 0; z-index: 60` (above sticky nav at z-50). `--color-bg` background. |
| Header | 48px height, "Close" button right-aligned. DM Sans 14px/600. Min-height 44px touch target on close button. |
| Content | Same `ChatInterface` content (heading, chips/messages, input, privacy notice). Full viewport height minus header. |
| Dismiss | Close button tap, browser back button, or swipe-down gesture. |
| Animation | Slide up from bottom, 300ms ease. Respects `prefers-reduced-motion` (instant show/hide). |
| Desktop | Not rendered. Desktop users see the inline chat container at section position 3. The inline container is always visible (no toggle needed). |

## Animation Contract (Phase 3 Scope)

Three new motions in Phase 3. All respect `prefers-reduced-motion`.

### Typing Dots Pulse

| Property | Value |
|----------|-------|
| Effect | Sequential opacity pulse: 0.3 -> 1.0 -> 0.3 per dot, staggered |
| Dot size | 8px diameter circles, `--color-accent` fill |
| Dot gap | 4px between dots |
| Stagger | 200ms delay between each dot (dot 1 at 0ms, dot 2 at 200ms, dot 3 at 400ms) |
| Cycle duration | 600ms total cycle, infinite repeat |
| Reduced motion | Static dots at full opacity, no animation. "Thinking..." text still visible. |

### Mobile Overlay Entrance

| Property | Value |
|----------|-------|
| Effect | `translateY(100%) -> translateY(0)` + backdrop opacity 0 -> 1 |
| Duration | 300ms |
| Easing | ease-out |
| Exit | Reverse: `translateY(0) -> translateY(100%)`, 200ms ease-in |
| Reduced motion | Instant show/hide (no translate, no fade) |

### Starter Chip Disappearance

| Property | Value |
|----------|-------|
| Effect | opacity 1 -> 0 + height collapse (heading + chips area) |
| Duration | 200ms |
| Easing | ease |
| Trigger | First user message sent |
| Reduced motion | Instant removal from DOM |

## Accessibility (extends Phase 1/2 baseline)

| Requirement | Phase 3 Implementation |
|-------------|------------------------|
| Focus rings | 2px solid `--color-accent`, 2px offset on input field, send button, starter chips, close button (inherited) |
| ARIA landmarks | Chat section: `role="complementary"` with `aria-label="Chat with Ryan"` (design doc spec). Message area: `role="log"` with `aria-live="polite"` for screen reader announcements of new messages. |
| Keyboard navigation | Tab order: starter chips (left to right) -> input field -> send button. After overlay opens: focus trapped within overlay. Esc closes overlay. |
| Touch targets | 44x44px minimum on: starter chips, send button, close button, mailto link in error state, calendar link in rate limit state |
| Screen reader | Starter chips: `aria-label="Suggested question: {chip text}"`. User messages: `aria-label="Your message"`. Assistant messages: `aria-label="Ryan's AI response"`. Typing indicator: `aria-label="Thinking..."` with `role="status"`. Send button: `aria-label="Send message"`. |
| Reduced motion | All 3 Phase 3 animations disabled (typing dots static, overlay instant, chips instant removal) |
| Input label | `aria-label="Type your message"` on the input field (no visible label -- placeholder serves as visual label) |
| Error announcement | Error and rate limit messages wrapped in `role="alert"` for immediate screen reader announcement |
